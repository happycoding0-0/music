# 🔐 Supabase 인증과 데이터베이스 연동 (Study Guide 05)

VibeTree의 사용자 로그인과 음악 포스팅이 어떻게 데이터베이스와 안전하게 대화하는지 알아봅니다.

---

## 🎯 1. 학습 목표

*   **인증(Auth)**과 **권한(RLS)**의 차이를 이해합니다.
*   Next.js에서 Supabase 클라이언트를 설정하고 호출하는 법을 익힙니다.

## 🛠️ 2. Supabase Auth: 로그인 시스템

기존에는 로그인을 직접 구현하려면 이메일 중복 체크, 비밀번호 암호화, 세션 토큰 발행 등 할 일이 태산이었습니다.  
하지만 **Supabase Auth**는 다음과 같은 일을 대신 해줍니다.

1.  **회원가입/로그인:** `supabase.auth.signUp()`과 `signInWithPassword()` 한 줄로 이메일/비밀번호 로그인이 가능합니다.
2.  **세션(Session) 관리:** 사용자가 나갔다 들어와도 브라우저에 쿠키나 로컬 스토리지를 자동으로 관리해서 "로그인 상태 유지"를 해줍니다.
3.  **사용자 식별:** 로그인 성공 시, 고유한 `user_id(UUID)`를 발급해 줍니다. 이 ID는 '게시글 작성자'를 구분하는 핵심 열쇠가 됩니다.

## 🗄️ 3. 데이터베이스와 테이블 설계

우리 앱의 심장인 `music_posts` 테이블은 아래와 같이 설계되어 있습니다.

*   `id`: 게시글의 고유 번호 (PK)
*   `user_id`: 글쓴이가 누구인가? (`auth.users` 테이블과 연결됨)
*   `title`, `artist`: 음악 정보
*   `youtube_url`: 실제 재생할 링크
*   `thumbnail_url`: 화면에 보여줄 앨범 아트

## 🛡️ 4. RLS (Row Level Security): 보안의 핵심

데이터베이스에 누구나 접근해서 남의 글을 지우면 안 되겠죠? 이럴 때 **RLS**라는 보안 정책을 사용합니다.

*   **모두에게 허용 (SELECT):** "로그인 안 한 사람도 노래 목록은 볼 수 있다."
*   **작성자만 허용 (INSERT/UPDATE/DELETE):** "글을 쓰거나 지우는 건 **로그인한 본인**만 할 수 있다."
*   이 설정 덕분에, 우리는 복잡한 백엔드 보안 코드를 짜지 않고도 데이터베이스 수준에서 안전하게 데이터를 보호할 수 있습니다.

## 💻 5. 코드 속에서의 흐름

1.  사용자가 '게시' 버튼을 누릅니다.
2.  `supabase.from('music_posts').insert([...])` 함수가 실행됩니다.
3.  Supabase 서버는 현재 로그인한 사용자의 토큰을 검사하여 글 쓸 권한이 있는지 확인합니다.
4.  권한이 있다면 DB에 저장하고 성공 신호를 돌려줍니다.
