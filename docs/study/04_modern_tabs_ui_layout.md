# 🧩 현대적인 탭(Tabs) 인터페이스 레이아웃 (Study Guide 04)

추천 곡과인기 차트를 번갈아 가며 볼 수 있는 세련된 탭 UI를 구현하고, 프론트엔드 배치 기술을 공부합니다.

---

## 🎯 1. 학습 목표

*   **Radix UI / Shadcn**을 활용한 표준적인 탭 컴포넌트 사용법을 익힙니다.
*   **상태 관리(State Management)**를 통해 현재 어떤 탭이 활성화되어 있는지 제어합니다.

## 🏗️ 2. 왜 Radix UI(Tabs)인가?

단순히 `div` 두 개를 놓고 `display: none`으로 가리고 보여주는 것보다, **Radix UI**의 탭 라이브러리를 쓰면 다음과 같은 장점이 있습니다.

1.  **접근성(Accessibility):** 키보드 화살표 키로 탭을 이동할 수 있고, 시각 장애인을 위한 스크린 리더도 완벽하게 지원합니다.
2.  **애니메이션:** 탭이 바뀔 때 부드럽게 전환되는 '마이크로 인터랙션'을 훨씬 쉽게 넣을 수 있습니다.
3.  **코드 구조:** '탭 제목'과 '탭 내용'이 명확하게 구분되어 코드가 매우 깔끔해집니다.

## 📱 3. 레이아웃 설계 (Layout Design)

### 3-1. 구조도
```
[ Header: MelodyShare ]
--------------------
[ Tab List: 추천곡 | 인기차트 ]
--------------------
[ Tab Content 1: 추천곡 목록 ]
  OR
[ Tab Content 2: 인기차트 목록 ]
--------------------
[ Global Music Player Bar ]
```

### 3-2. 반응형(Responsive) 고려
*   **데스크톱:** 한 줄에 3~4개의 카드를 보여주는 그리드 형태.
*   **모바일:** 화면이 좁으므로 한 줄에 1개 또는 2개의 카드로 자동 조정. (Flexbox/Grid 활용)

## 💡 4. 데이터 페칭(Data Fetching) 전략

*   **추천곡(Curated):** 서버 사이드 렌더링(SSR) 또는 서버 컴포넌트에서 직접 Supabase DB를 조회하여 즉시 보여줍니다.
*   **인기차트(Popular):** 외부 API를 부르는 시간이 걸리므로, 탭을 눌렀을 때 클라이언트에서 비동기(`fetch`)로 가져오거나 로딩 상태(Skeleton)를 보여주는 것이 좋습니다.

## 📝 5. 오늘의 핵심 키워드

1.  **Skeleton UI:** 데이터를 불러오는 동안 가짜 회색 박스를 보여주어 사용자가 "아, 로딩 중이구나"라고 느끼게 하는 기법.
2.  **Conditional Rendering:** `{ activeTab === 'popular' && <PopularFeed /> }` 와 같이 조건에 따라 화면을 다르게 그리는 기술.
