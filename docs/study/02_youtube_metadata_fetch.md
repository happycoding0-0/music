# 🔗 유튜브 링크로 데이터 자동 완성하기 (Study Guide 02)

사용자가 URL만 입력했는데 제목과 가수 이름이 자동으로 채워지는 기능은 UX(사용자 경험)에서 매우 중요합니다. 이를 가능하게 하는 **oEmbed** 기술에 대해 알아봅니다.

---

## 💡 1. oEmbed란 무엇인가?

**oEmbed**는 웹사이트(A)가 다른 웹사이트(B)의 콘텐츠(유튜브 영상, 트위터 게시물 등)를 내 페이지에 쉽게 임베드(삽입)하거나, 그 주소에 대한 정보를 가져올 수 있게 설계된 **개방형 표준 프로토콜**입니다.

*   **기존 방식:** 내가 직접 유튜브 페이지를 크롤링(HTML을 다 긁어옴)해서 제목을 찾아야 함. (매우 느리고 차단될 위험 높음)
*   **oEmbed 방식:** 유튜브가 공식적으로 제공하는 "정보 창구"에 주소만 던져주면, 유튜브가 정제된 데이터(제목, 작성자, 썸네일 등)를 예쁘게 포장해서 돌려줌.

## 🛠️ 2. 데이터 흐름 (Data Flow)

1.  **사용자 동작:** 사용자가 `Input`창에 유튜브 주소를 붙여넣습니다.
2.  **클라이언트 (Next.js):** 주소가 올바른지 확인한 후, 우리 서버 API(`/api/youtube/metadata`)로 주소를 전달합니다.
3.  **서버 (API Route):** 서버는 유튜브의 oEmbed API 주소에 요청을 보냅니다.
    *   요청 주소 예시: `https://www.youtube.com/oembed?url=https://www.youtube.com/watch?v=...&format=json`
4.  **응답:** 유튜브 서버가 아래와 같은 데이터를 돌려줍니다.
    ```json
    {
      "title": "BTS (방탄소년단) 'Dynamite' Official MV",
      "author_name": "HYBE LABELS",
      "thumbnail_url": "...",
      "type": "video"
    }
    ```
5.  **화면 반영:** 받아온 `title`을 제목 칸에, `author_name`을 아티스트 칸에 자동으로 쏙 넣어줍니다.

## ⚡ 3. 왜 직접 안 부르고 서버를 거치나요? (CORS 이슈)

브라우저(클라이언트)에서 유튜브 oEmbed API로 직접 데이터를 요청하면, 보안상의 이유로 **CORS(Cross-Origin Resource Sharing)** 에러가 발생하며 차단될 수 있습니다. 
이를 해결하기 위해 **'Proxy(대리인)'** 역할을 하는 우리 서버 API를 거쳐서 데이터를 가져오는 것이 실무의 정석입니다.

## 📝 4. 오늘의 학습 포인트

1.  **필수 해제:** 과거에는 '설명'을 필수로 적게 했지만, 이제는 제목과 아티스트가 자동으로 채워지므로 사용자의 입력을 최소화하기 위해 '설명' 칸은 **Optional(선택 사항)**로 변경합니다.
2.  **정규식 (Regex):** URL에서 `v=ID` 값을 뽑아내는 정규식을 통해 올바른 링크인지 한 번 더 검증하는 과정을 거칩니다.
